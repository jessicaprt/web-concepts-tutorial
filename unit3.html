XML

  • Extensible Markup Language (XML) is a way to create a markup of any kind of information in a human readable way
  • Example:
  • <xml version="1.0"> <!-- info obtained from imdb --> <movie>    <title>Inception</title>    <releaseDate>18 July 2010</releaseDate>    <director>Christopher Nolan</director>    <writer>Christopher Nolan</writer> </movie>

Document Type Definitions (DTD)

  • uses EBNF (Extended Backus-Naur Form) grammar that specifies the document's element types and their relationship with each other.
  • DTD verifies XML documents.
  • DTD files are saved as .dtd
  • Example:
  • --> sample code here

Schema Documents

  • Schema documents specify an element's data type
  • These documents use XML syntax
  • Schema documents are saved as a .xsd file
  • some types that can be used:
    • string
    • boolean
    • decimal
    • float
    • double
    • long
    • int
    • short
    • date
    • time
  • There are two types of XML schema: simple type and complex types
    • The difference is that complex types can hold child elements.
  • Example:
  • <?xml version="1.0"?>
    <schema xmlns="http://www.w3.org/2001/XMLSchema"
      xmlns:resume = "http://jessica.prieto.ca/resume"
      targetNamespace = "http://jessica.prieto.ca/resume">
       <element name="my-resume" type="ResumeType"/>
       <complexType name="ResumeType">
         <sequence>
           <element name="name" type="string"/>
            <element name="contactInfo" type="string"
             minOccurs="1" maxOccurs="unbounded"/>
           <element name="section" type="sectionType"/>
         </sequence>
       </complexType>

       <complexType name = "sectionType">
         <sequence>
           <element name = "subsection" type="subsections"
             minOccurs = "0" maxOccurs = "unbounded"/>
           <element name = "bullet-point" type="string"
             minOccurs = "0" maxOccurs = "unbounded"/>
         </sequence>
       </complexType>

       <complexType name = "subsections">
         <sequence>
           <element name = "title" type="string"
             minOccurs = "0" maxOccurs = "1"/>
           <element name = "description" type="string"
             minOccuts = "0" maxOccurs = "1"/>
           <element name = "date" type="string"
             minOccuts = "0" maxOccurs = "1"/>
           <element name = "details" type="string"
             minOccuts = "0" maxOccurs = "unbounded"/>
         </sequence>
       </complexType>
    </schema>

Extensible Style Sheet (XSL)

  • XSL converts XML into any text-based document
  • Just like CSS to HTML, XSL can also be embeded in an XML file by adding:
  • <xml:stylesheet>
  • there are two important tree structures involved:
    • source tree
    • result tree tree

  • Some important tools to remember:

    tool/ function description
    / selects the document's root
    value-of retrieves the value of the element
    node-set function name retrieves the node's element name
    node-set function text retrieves the text between the start and end tags
    //* selects all nodes in the document

  • Documented Object Model in XML

AJAX

  • Ajax applications separate client-side user interaction and server communication
  • It still uses JavaScript to send requests (i.e., HTTP requests) to the server and then updates the pages using DOM
  • AJAX also uses HTML and CSS for the web page front-end with some functionality with JS. XML and JSON is used for the data that’s passed around in the requests.

  • some AJAX request methods

    method description
    readystatechange creates a listener that listens when the server responds
    readyState keeps track of the progress
    responseText server's response to the client (in text format)
    responseXML server's response to the client (in XML format)
    status gets the HTTP Status: some examples of return values: 200 - OK, 400 - Bad request, 401 - unauthorized, 403 - forbidden, 404 - not authorized, etc...
    statusText a more descriptive information of the response status
    open initializes the HTTP resuest. some examples: GET - asks for data from the server, PUT - sends data to the server, POST - changes the data in the server
    send sends the HTTP request
    setRequestHeader changes the request header
    getRequestHeader gets the data from the request header
    getAllResponseHeader returns an array of the header data
    abort aborts the request
